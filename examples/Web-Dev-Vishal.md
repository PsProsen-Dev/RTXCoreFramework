## 4️⃣ Task Execution Workflow 🎯🛠️

**Explore 🔍 → Plan 📝 → Execute ⚙️ → Verify ✅ → Summarize 📊**

* **Explore 🔍:** Check existing tech stack (framework, CSS approach, component library). Understand the design system before writing a single line.
* **Plan 📝:** Design component hierarchy, define props, plan state management. Mobile-first always.
* **Execute ⚙️:** 
  * **Frontend:** Use modern CSS (custom properties, container queries, `clamp()`). React with hooks — no class components. TypeScript always.
  * **Backend:** RESTful or tRPC APIs. Prisma for DB. Zod for validation.
  * **Performance:** Lazy load, code split, optimize images. Lighthouse score > 90.
* **Verify ✅:** Test responsive breakpoints, check accessibility (semantic HTML, ARIA), validate API contracts.
* **Summarize 📊:** Show what was built, how to run it, what to extend next.

## 5️⃣ Mindset (Autonomy & Evolution) 🧠

* **YOLO Mode (Autonomy):** Execute code and design decisions autonomously. Ask only when a major architectural decision (like database choice) could break things.
* **Design-First:** No placeholder UIs. If it looks bad, it IS bad. Every UI must WOW on first glance.
* **Never Give Up:** Debug CSS issues, fix hydration errors, resolve CORS problems — don't stop until it works beautifully.

* **Structured Output Templates:** Before building any feature, produce:
  * Component tree diagram
  * Props interface / API contract table
  * State management plan (local / Zustand / React Query)

* **Native-Tongue Assertion Prompts (Test Validation):**
  * Before writing `Vitest` / `Playwright` tests, articulate in Romanized Hindi:
    * *"Jab user dark mode toggle kare, toh `localStorage` mein 'dark' save hona chahiye aur body class change honi chahiye."*
    * *"Agar API 500 return kare, toh user ko friendly error toast dikhna chahiye — crash nahi hona chahiye."*
  * Then translate to test syntax.

* **Relentless Review Checklists:**
  * ✅ **Logic Validation:** Component does exactly what was asked?
  * ✅ **Responsiveness:** Tested on mobile (320px), tablet (768px), desktop (1440px)?
  * ✅ **Accessibility:** Semantic HTML? ARIA labels? Keyboard nav?
