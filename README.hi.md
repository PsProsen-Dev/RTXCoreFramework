## 🚀 (RTX⚡) Core Framework Kya Hai?

**(RTX⚡) Core Framework** (Reasoning | Thinking | Xtreme) ek universal AI agent framework hai jo kisi bhi AI assistant, LLM, ya agentic platform pe globally run kar sakta hai. Yeh ek master blueprint hai jo generic AI behavior ko highly autonomous, personalized aur execution-focused system mein transform karta hai.

**RTX ka matlab:**
- **R – Reasoning:** Socho, logic lagao, user ka intent samjho
- **T – Thinking:** Continuously seekho, khud ko improve karo
- **X – Xtreme:** Fast action lo, maximum productivity lo

---

## 🌟 Vision: Language Barrier Todna

Programming aur software development ka entry barrier khatam karna — **yahi RTX ka core mission hai.**

* **Tech ko Democratize karo:** Language kabhi barrier nahi banni chahiye. Duniya ke kisi bhi kone se koi bhi apni **maa-boli mein AI ke saath kaam karke** outstanding software bana sakta hai.

* **"Syntax Master" se "Theoretical Architect" bano:** Users ko programming languages, CLI tools, ya dependencies master karne ki zarurat nahi. Sirf samjho ki component **kya** karta hai, **kaise** karta hai, aur **kyun** — phir apni maa-boli mein agent ko bolo aur software build ho jayega.

* **Jensen Huang ka jawab:** NVIDIA ke CEO ne kaha — *"English, future ki programming language hai."* **Hum strongly disagree karte hain.** Sirf English tak limit karna 5 billion non-English speakers ko bahar rakh deta hai. Future ki programming language aapki **apni maa-boli** hai.

* **70% Romanized + 30% English ka blend kyun?** Kyunki technical docs, code, aur Stack Overflow sab English mein hain — isliye pure native script kaam nahi karti. Romanized blend dono worlds ka best combine karta hai. Hindi bolne wala "Agar user authenticated nahi hai, 401 throw karo" likh sakta hai — aur agent samjhega!

## 🎯 Kaise Use Karein — Zero Prompt Required

> **Golden Rule:** Sirf **ek file** chahiye — `RTXCoreFramework.md`. Isko apne agent ko ek baar do — baki sab agent khud handle kar leta hai, hamesha ke liye.

**Step 1 — File download karo:**
👉 [`RTXCoreFramework.md`](https://raw.githubusercontent.com/PsProsen-Dev/RTXCoreFramework/master/framework/RTXCoreFramework.md) — Right-click → Save As

**Step 2 — Agent ko file do:**

| Tool Type | Kaise Do |
|-----------|---------|
| **Koi bhi AI Chat / IDE** | `.md` file ko chat mein drag & drop karo, ya 📎 attach button use karo |
| **Agentic CLI** | Terminal kholo → CLI launch karo → Chat mein sirf file path paste karo |
| **Code Editor** | File ko global rules directory mein rakh do aur restart karo |

> **File dene ke baad — kuch aur type mat karo.** Agent file padh ke apne aap First-Boot Protocol shuru kar deta hai.

## 🤖 File Dene Ke Baad Kya Hota Hai?

Agent automatically:

**1.** **3 quick setup questions** poochega *(Maa-boli → Agent ka Naam → Aapko kaise address karein)*

**2.** Apna OS detect karega (Windows, Mac, ya Linux) aur **bina permission maange** framework ko system mein inject kar dega

**3.** **(RTX⚡)** framework ko system ke **sabhi installed AI tools** mein propagate karega

**4.** Ek **Background OS Hook** register karega — taaki future mein install hone wala koi bhi naya AI tool automatically RTX framework pa sake

```
RTXCoreFramework.md kisi BHI EK agent ko do  (sirf ek baar)
                        ↓
         Agent 3 setup questions poochega
                        ↓
    Agent OS detect karega & globally inject karega
                        ↓
   ✅ Antigravity  ✅ Codex  ✅ OpenCode  ✅ Claude
   ✅ Copilot      ✅ Cursor  ✅ Sabhi future tools
                        ↓
    Dobara kuch karne ki zarurat nahi. Kabhi nahi.
```

## ❓ Troubleshooting

**Agent pure English mein bol raha hai?**
→ Naya chat shuru karo. Agar phir bhi ho toh type karo: `"RTX Anti-Drift — 70/30 blend restore karo abhi."`

**3 questions trigger nahi hue?**
→ File content copy karo aur tool ke Custom Instructions / System Prompt mein paste karo, phir naya chat shuru karo.

**Cursor ya Codex ignore kar raha hai?**
→ `RTXCoreFramework.md` ko `~/.cursor/rules/` ya `~/.codex/AGENTS.md` mein manually rakh do.

**Uninstall karna hai?**
→ `README.md` mein [uninstall guide](README.md#-troubleshooting--faq) dekho — saari paths listed hain.
